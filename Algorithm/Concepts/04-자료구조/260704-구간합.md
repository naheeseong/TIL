# 4-2 구간 합 (Prefix Sum)

## 1. 구간 합 구하기

### 1.1. 개념 및 사용 조건
- 공식: `S[i] = S[i-1] + A[i]`
- 트리거: 데이터의 변경이 없고(Immutable), "연속된 구간의 합을 반복해서(여러 번) 구해야 할 때" 무조건 꺼내 쓴다.
- 배열 설계: 범위를 구할 때 `left-1` 처리를 해야 하므로, 인덱스 에러 방지를 위해 배열 크기는 무조건 `N + 1`로 잡고 0번 칸은 `0`으로 비워둔다.

### 1.2. 성능 분석
- 시간 복잡도:
  - 단순 반복문 누적: 구간 합을 구할 때마다 루프를 돌아서 $O(N)$
  - 구간 합 배열 활용: 이미 더해둔 값을 한 번만 빼면 되므로 $O(1)$ (상수 시간) 만에 해결 가능.
  - 질의가 $M$번 들어올 때 전체 시간 복잡도를 $O(N \times M)$에서 $O(N + M)$으로 획기적으로 줄임.

### 1.3. 주의
- 인덱스 에러: 배열 크기를 N으로 잡고 `sumRange`에서 `left - 1`을 했다가 `left = 0`일 때 `ArrayIndexOutOfBoundsException` 발생함. 반드시 $N+1$ 크기 패턴 유지할 것.
- 자바 캡슐화: 릿코드 채점 서버는 클래스를 내부적으로 인스턴스화해서 호출하므로, 멤버 변수인 누적 합 배열은 `private`으로 선언하여 외부 오염을 막아야 함.

### 1.4. 핵심 로직 (코드 힌트)
```java
// 1번 인덱스부터 채우는 정석 흐름 (nums의 값을 하나씩 당겨와 매칭)
for (int i = 1; i <= n; i++) {
    prefixSum[i] = prefixSum[i - 1] + nums[i - 1]; 
}
```

## 2. 구간 합 구하기2


### 2.1. 개념 및 사용 조건
- 공식: 2D 누적합 배열 구축 시 `S[i][j] = S[i-1][j] + S[i][j-1] - S[i-1][j-1] + A[i][j]`
- 트리거: 2D 배열에서 데이터 변경이 없고(Immutable), "특정 직사각형 영역의 합을 반복해서(여러 번) 구해야 할 때" 사용
- 배열 설계: 2D 배열이므로 행과 열 모두 N+1로 설계하고 0번 행과 0번 열은 0으로 비워둔다. 이는 sumRegion 계산 시 경계 케이스(row1=0, col1=0)를 안전하게 처리하기 위함.

### 2.2. 성능 분석
- 시간 복잡도:
  - 단순 중첩 반복문 누적: 구간 합을 구할 때마다 (row2-row1) * (col2-col1) 번 루프를 돌아서 최악의 경우 O(N^2)
  - 2D 누적합 배열 활용: 미리 계산된 값 4개를 더하고 빼는 것만으로 O(1) (상수 시간) 만에 해결 가능.
  - 질의가 M번 들어올 때 전체 시간 복잡도를 O(N^2 * M)에서 O(N^2 + M)으로 획기적으로 줄임.

### 2.3. 주의
- 인덱스 에러: 배열 크기를 (rowCount) x (colCount)로 잡으면 sumRegion에서 row1-1, col1-1을 할 때 0일 경우 마이너스 인덱스 에러(ArrayIndexOutOfBoundsException) 발생. 반드시 (rowCount+1) x (colCount+1) 크기 패턴 유지할 것.
- 포함-배제 원리: sumRegion에서 빼기를 두 번, 더하기를 한 번 하는 이유는 겹치는 영역을 정확히 제거하기 위함. 공식의 순서를 바꾸면 틀린 값이 나옴.
- 자바 캡슐화: 리트코드 채점 서버는 클래스를 내부적으로 인스턴스화해서 호출하므로, 멤버 변수는 private으로 선언하여 외부 오염을 막아야 함.

### 2.4. 핵심 로직 (코드 힌트)
```java
// 1번 행과 1번 열부터 채우는 정석 흐름 (matrix의 값을 하나씩 당겨와 매칭)
for(int i=1;i<=rowCount;i++){
    for(int j=1;j<=colCount;j++){
        prefixSum[i][j] = prefixSum[i-1][j] + prefixSum[i][j-1] 
                          - prefixSum[i-1][j-1] + matrix[i-1][j-1];
    }
}

// 직사각형 구간 합 쿼리 (포함-배제 원리)
return prefixSum[row2+1][col2+1] - prefixSum[row1][col2+1] 
       - prefixSum[row2+1][col1] + prefixSum[row1][col1];
```

## 3. 나머지 합 구하기

### 3.1. 개념 및 사용 조건
- 공식: 같은 나머지를 가진 두 누적합 위치를 찾아 그 사이의 구간 합을 구한다
- 핵심: S[i] % k == S[j] % k이면 (S[i] - S[j]) % k = 0, 즉 j+1부터 i까지의 구간 합이 k의 배수
- 트리거: 배열의 "특정 구간 합이 특정 수로 나누어떨어지는 경우의 수"를 구할 때 사용
- 데이터: 배열이 변경되지 않는 Immutable 상태여야 함

### 3.2. 핵심 아이디어
누적합 배열 D를 0부터 N-1까지 순회하면서 각 D[i] % k의 나머지를 HashMap(또는 배열) C에 저장한다. 같은 나머지가 나타나면 그 사이의 구간이 k의 배수라는 뜻이므로, 조합 공식 nC2 = n*(n-1)/2를 이용해 경우의 수를 계산한다.

### 3.3. 주의 - 음수 나머지 처리
자바에서 음수를 나머지 연산하면 음수가 나온다.
- 예: -2 % 5 = -2 (양수가 아님)
- 해결: tmp < 0이면 tmp += k로 양수로 변환
- 이유: 배열 인덱스는 음수를 허용하지 않으므로 범위를 벗어나는 에러 발생

### 3.4. 성능 분석
- 시간 복잡도: O(N) - 누적합 계산 + 나머지 카운팅 + 조합 계산 모두 선형
- 공간 복잡도: O(k) - 나머지 카운팅 배열의 크기는 k
- 질의가 여러 번 들어와도 O(N)에 한 번만 계산하면 됨

### 3.5. 핵심 로직
```java
// 1단계: 누적합 배열 구축
for (int i = 1; i < N; i++){
    D[i] = D[i-1] + nums[i];
}

// 2단계: 나머지 카운팅 및 직접 카운트
for(int i = 0; i < N; i++){
    int tmp = D[i] % k;
    
    // 음수 나머지 처리
    if (tmp < 0){
        tmp += k;
    }
    
    // 0으로 나누어떨어지는 경우 직접 카운트
    if (tmp == 0){
        answer++;
    }
    
    // 나머지별 개수 저장
    C[tmp]++;
}

// 3단계: 같은 나머지를 가진 쌍의 조합 계산
for (int i = 0; i < k; i++){
    answer += (C[i] * (C[i] - 1)) / 2;
}
```

### 3.6. 예시 추적
nums = [4, 5, 0, -2, -3, 1], k = 5

누적합: D = [4, 9, 9, 7, 4, 5]
나머지: [4, 4, 4, 2, 4, 0]

나머지별 개수: C[0]=1, C[2]=1, C[4]=4

결과:
- tmp == 0인 경우: 1 (인덱스 5)
- 같은 나머지 4를 가진 조합: 4C2 = 4*(4-1)/2 = 6
- 총 경우의 수: 1 + 6 = 7
