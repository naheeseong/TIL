# 4-3 투 포인터

## 1. 연속된 자연수의 합 구하기

### 1.1. 개념 및 사용 조건

공식: `start = (n - k(k-1)/2) / k` (k: 연속된 정수의 개수)

트리거: 정수 n을 "연속된 양의 정수의 합으로 나타내는 경우의 수"를 구할 때 사용

핵심: 각 k에 대해 `remainder % k == 0`이면 start가 양의 정수가 되어 유효한 분해가 존재함을 의미

### 1.2. 성능 분석

시간 복잡도:
- Sliding Window 방식: 모든 경우를 순회하면 O(N) → n이 10^9일 때 TLE 발생
- 수학적 최적화: k(k-1)/2 < n 조건으로 약 √(2n) 번만 반복 → O(√n)

공간 복잡도: O(1) - 상수 공간만 사용

질의가 한 번 들어올 때 전체 시간 복잡도를 O(N)에서 O(√n)으로 획기적으로 줄임

### 1.3. 주의

인덱스 조건: 루프 범위는 `k(k-1)/2 < n`이어야 함
- `k(k-1)/2 >= n`이면 `start <= 0`이 되므로 더 이상 확인할 필요 없음
- 이는 k가 약 √(2n) 정도에서 자동으로 종료됨을 의미

오버플로우 방지: `k * (k - 1) / 2`와 remainder 계산 시 `long` 타입 사용 권장
- 특히 n이 10^9일 때 정수 오버플로우 가능

### 1.4. 핵심 로직 (코드 힌트)

// 각 k에 대해 start가 정수인지 확인하는 로직
for (int k = 2; k * (k - 1) / 2 < n; k++) {
long remainder = n - (long)k * (k - 1) / 2;

    // remainder가 k로 나누어떨어지면 start는 정수
    if (remainder % k == 0) {
        count++;
    }
}

### 1.5. 예시 추적

n = 15

| k | k(k-1)/2 | remainder | remainder%k==0? | start | 표현 |
|---|----------|-----------|-----------------|-------|------|
| 1 | - | - | - | - | 15 |
| 2 | 1 | 14 | 14%2=0 ✓ | 7 | 7+8 |
| 3 | 3 | 12 | 12%3=0 ✓ | 4 | 4+5+6 |
| 4 | 6 | 9 | 9%4=1 ✗ | - | - |
| 5 | 10 | 5 | 5%5=0 ✓ | 1 | 1+2+3+4+5 |
| 6 | 15 | 0 | 0%6=0 ✓ | 0 | ✗ (start는 1 이상이어야 함) |

결과: k=1,2,3,5에서만 유효 → count = 4
