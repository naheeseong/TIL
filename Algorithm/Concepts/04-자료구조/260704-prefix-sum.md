# 4-2 구간 합 (Prefix Sum)

### 1. 개념 및 사용 조건
- 공식: `S[i] = S[i-1] + A[i]`
- 트리거: 데이터의 변경이 없고(Immutable), "연속된 구간의 합을 반복해서(여러 번) 구해야 할 때" 무조건 꺼내 쓴다.
- 배열 설계: 범위를 구할 때 `left-1` 처리를 해야 하므로, 인덱스 에러 방지를 위해 배열 크기는 무조건 `N + 1`로 잡고 0번 칸은 `0`으로 비워둔다.

### 2. 성능 분석
- 시간 복잡도:
  - 단순 반복문 누적: 구간 합을 구할 때마다 루프를 돌아서 $O(N)$
  - 구간 합 배열 활용: 이미 더해둔 값을 한 번만 빼면 되므로 $O(1)$ (상수 시간) 만에 해결 가능.
  - 질의가 $M$번 들어올 때 전체 시간 복잡도를 $O(N \times M)$에서 $O(N + M)$으로 획기적으로 줄임.

### 3. 주의
- 인덱스 에러: 배열 크기를 N으로 잡고 `sumRange`에서 `left - 1`을 했다가 `left = 0`일 때 `ArrayIndexOutOfBoundsException` 발생함. 반드시 $N+1$ 크기 패턴 유지할 것.
- 자바 캡슐화: 릿코드 채점 서버는 클래스를 내부적으로 인스턴스화해서 호출하므로, 멤버 변수인 누적 합 배열은 `private`으로 선언하여 외부 오염을 막아야 함.

### 4. 핵심 로직 (코드 힌트)
```java
// 1번 인덱스부터 채우는 정석 흐름 (nums의 값을 하나씩 당겨와 매칭)
for (int i = 1; i <= n; i++) {
    prefixSum[i] = prefixSum[i - 1] + nums[i - 1]; 
}
