# 스택 트레이스 읽기

구조

```
Exception in thread "main" java.util.NoSuchElementException
                        ↑ 에러 타입

	at java.base/java.util.StringTokenizer.nextToken(StringTokenizer.java:349)
	at Main.main(Main.java:19)
	↑ 아래에서 위로 읽음
```



### 1단계: 에러 타입 파악
| 에러 | 의미 |
|-----|-----|
| `NoSuchElementException` | 요소가 없음 |
| `NullPointerException` | null 값 사용 |
| `ArrayIndexOutOfBoundsException` | 배열 범위 초과 |
| `NumberFormatException` | 숫자 변환 실패 |
| `StringIndexOutOfBoundsException` | 문자열 범위 초과 |

### 2단계: 스택 트레이스 읽기 (아래→위)

```
Main.java:19           ← 내 코드 (여기서 시작!)
    ↑
StringTokenizer.java   ← 라이브러리 (참고만)
    ↑
NoSuchElementException ← 최종 원인
```

핵심: `at Main.java:19` 부분이 **내가 봐야 할 곳**

예시: NoSuchElementException

```java
// Main.java:19
StringTokenizer st = new StringTokenizer("5 3");
int a = st.nextToken();  // "5" ✓
int b = st.nextToken();  // "3" ✓
int c = st.nextToken();  // ← 토큰 없음! NoSuchElementException 터짐
```

해결:
```java
if (st.hasMoreTokens()) {
    String token = st.nextToken();
}
```

 IDE 활용

- 스택 트레이스에서 `Main.java:19` 같은 부분을 클릭
- IDE가 자동으로 그 줄로 이동
- 문제 원인을 즉시 확인 가능


### 요약

```
Exception 타입 → 뭔지 파악
스택 트레이스 → 아래에서 위로 읽음
Main.java:XX → 내 코드는 여기!
```
